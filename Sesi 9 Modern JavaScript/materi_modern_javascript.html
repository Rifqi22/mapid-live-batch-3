1. Apa itu Modern JavaScript? Sesi 9

Modern JavaScript mengacu pada fitur-fitur yang diperkenalkan sejak ES6 (ECMAScript 2015) dan versi setelahnya. Fitur ini membuat kode lebih singkat, mudah dibaca, dan aman.

💡 ECMAScript (ES) adalah standar resmi bahasa JavaScript. ES6 (2015) adalah update terbesar yang mengubah cara kita menulis JS. Versi setelahnya (ES2016, ES2017, ...) menambahkan fitur baru setiap tahun.

Kenapa Belajar Modern JavaScript?

ManfaatContoh
Kode lebih singkatArrow function, destructuring
Lebih mudah dibacaTemplate literal, async/await
Lebih aman dari errorOptional chaining, nullish coalescing
Standar industriHampir semua kode & library modern memakainya
Wajib untuk frameworkReact, Vue, Node.js semua pakai ES6+

2. let & const — Block Scope ES6

Di JavaScript lama, hanya ada var. ES6 memperkenalkan let dan const yang lebih aman karena memiliki block scope.

Aspekvar (lama)let / const (modern)
ScopeFunction scopeBlock scope { }
Re-deklarasi✅ Boleh (rawan bug)❌ Tidak boleh
Hoisting✅ (jadi undefined)❌ (error jika dipakai duluan)
Rekomendasi❌ Hindari✅ Selalu pakai ini
// Block scope: let & const hanya hidup di dalam { } if (true) { let pesan = "Halo"; console.log(pesan); // "Halo" ✅ } // console.log(pesan); ← ERROR! pesan tidak ada di luar blok // var "bocor" keluar dari blok (sumber bug!) if (true) { var bocor = "Aku bocor"; } console.log(bocor); // "Aku bocor" ← var tidak peduli blok!
✅ Aturan praktis: Default pakai const. Pakai let hanya jika nilai perlu berubah. Jangan pernah pakai var.

3. Arrow Function (Mendalam) ES6

Arrow function adalah cara singkat menulis function. Sangat sering dipakai, terutama di dalam array methods dan callback.

Evolusi Penulisan Function

// 1. Function declaration (cara lama) function tambah(a, b) { return a + b; } // 2. Arrow function dengan body { } const tambah = (a, b) => { return a + b; }; // 3. Arrow function singkat (implicit return — tanpa { } dan return) const tambah = (a, b) => a + b; // 4. Satu parameter — kurung boleh dihilangkan const kuadrat = n => n * n; // 5. Tanpa parameter — kurung wajib kosong const sapa = () => "Halo Dunia"; console.log(tambah(3, 4)); // 7 console.log(kuadrat(5)); // 25

Implicit Return untuk Object

// Jika ingin return OBJECT langsung, bungkus dengan kurung ( ) const buatLokasi = (nama, kota) => ({ nama: nama, kota: kota }); console.log(buatLokasi("Monas", "Jakarta")); // { nama: "Monas", kota: "Jakarta" } // Tanpa kurung ( ), { } dianggap body function → error/undefined

Kapan Pakai Arrow Function?

Function biasa
arr.forEach(function(x) {
  console.log(x);
});
Arrow (lebih ringkas)
arr.forEach(x => {
  console.log(x);
});
⚠️ Catatan: Arrow function tidak punya this sendiri (mewarisi dari luar). Untuk materi dasar, cukup tahu arrow function sangat cocok untuk callback singkat seperti di map, filter, forEach.

4. Destructuring ES6

Destructuring adalah cara cepat "membongkar" isi array atau object ke dalam variabel. Sangat berguna untuk data GeoJSON!

A. Array Destructuring

// Cara lama const koordinat = [107.61, -6.90]; const lng = koordinat[0]; const lat = koordinat[1]; // Cara modern — destructuring (urutan sesuai posisi) const [lng, lat] = [107.61, -6.90]; console.log(lng); // 107.61 console.log(lat); // -6.90 // Contoh nyata di GeoJSON: const feature = { geometry: { coordinates: [107.68, -6.90] } }; const [bujur, lintang] = feature.geometry.coordinates;

B. Object Destructuring

const lokasi = { nama: "Apartemen Transit", kota: "Bandung", status: "BUKA" }; // Cara lama const nama = lokasi.nama; const kota = lokasi.kota; // Cara modern — destructuring (nama variabel = nama key) const { nama, kota, status } = lokasi; console.log(nama); // "Apartemen Transit" console.log(kota); // "Bandung" // Ganti nama variabel dengan : const { nama: namaLokasi } = lokasi; console.log(namaLokasi); // "Apartemen Transit" // Nilai default jika key tidak ada const { telepon = "-" } = lokasi; console.log(telepon); // "-" (karena lokasi tidak punya telepon)

C. Destructuring di Parameter Function

// Langsung "bongkar" object di parameter function tampilLokasi({ nama, kota, status }) { return `${nama} di ${kota} — ${status}`; } const data = { nama: "Monas", kota: "Jakarta", status: "BUKA" }; console.log(tampilLokasi(data)); // "Monas di Jakarta — BUKA"
✅ Sangat berguna untuk GeoJSON: const { NAMA, KABKOT, STATUS } = feature.properties; langsung mengambil 3 properti sekaligus dalam satu baris.

5. Spread & Rest Operator (...) ES6

Tanda tiga titik ... punya 2 fungsi tergantung konteks: Spread (memecah) dan Rest (mengumpulkan).

A. Spread — Memecah / Menyalin

// Menggabungkan array const jakarta = ["Monas", "Ancol"]; const bandung = ["Dago", "Lembang"]; const semua = [...jakarta, ...bandung]; console.log(semua); // ["Monas", "Ancol", "Dago", "Lembang"] // Menyalin array (copy, bukan referensi) const salinan = [...jakarta]; // Menggabungkan / menyalin object const lokasi = { nama: "Monas", kota: "Jakarta" }; const lokasiLengkap = { ...lokasi, status: "BUKA" }; console.log(lokasiLengkap); // { nama: "Monas", kota: "Jakarta", status: "BUKA" } // Update sebagian property (override) const updated = { ...lokasi, kota: "Bandung" }; // { nama: "Monas", kota: "Bandung" }

B. Rest — Mengumpulkan Sisa

// Rest di destructuring array const [pertama, ...sisanya] = ["TJ", "MRT", "LRT", "KRL"]; console.log(pertama); // "TJ" console.log(sisanya); // ["MRT", "LRT", "KRL"] // Rest di parameter function (jumlah argumen bebas) function jumlahkan(...angka) { return angka.reduce((total, n) => total + n, 0); } console.log(jumlahkan(1, 2, 3)); // 6 console.log(jumlahkan(10, 20, 30, 40)); // 100
💡 Cara mengingat: ... di sisi kanan (saat membuat array/object) = Spread (memecah). ... di sisi kiri (saat destructuring/parameter) = Rest (mengumpulkan).

6. Default Parameter, Ternary & Short-circuit ES6

A. Default Parameter

// Beri nilai default jika argumen tidak dikirim function sapa(nama = "Tamu", kota = "Indonesia") { return `Halo ${nama} dari ${kota}`; } console.log(sapa("Rifqi", "Bandung")); // "Halo Rifqi dari Bandung" console.log(sapa("Andi")); // "Halo Andi dari Indonesia" console.log(sapa()); // "Halo Tamu dari Indonesia"

B. Ternary Operator (if/else singkat)

// Struktur: kondisi ? nilai_jika_true : nilai_jika_false const nilai = 85; // Cara lama let hasil; if (nilai >= 70) { hasil = "Lulus"; } else { hasil = "Tidak Lulus"; } // Cara modern — ternary (1 baris) const hasil = nilai >= 70 ? "Lulus" : "Tidak Lulus"; console.log(hasil); // "Lulus" // Contoh WebGIS: badge status const status = "BUKA"; const warna = status === "BUKA" ? "green" : "red";

C. Short-circuit (|| dan &&)

// || (OR) — pakai nilai cadangan jika kiri "falsy" (kosong/null/undefined) const nama = ""; const tampil = nama || "Tanpa Nama"; console.log(tampil); // "Tanpa Nama" // Sangat berguna untuk data API yang kadang kosong: const telepon = data.TELEPON || "Tidak tersedia"; // && (AND) — jalankan kanan hanya jika kiri "truthy" const user = { nama: "Rifqi" }; user && console.log(user.nama); // "Rifqi" (karena user ada)

7. Optional Chaining & Nullish Coalescing ES2020

A. Optional Chaining (?.)

Mencegah error saat mengakses property dari object yang mungkin undefined/null. Sangat sering dipakai untuk data API!

const feature = { properties: { NAMA: "Monas" } // catatan: tidak ada "geometry" }; // ❌ Cara lama — ERROR jika geometry tidak ada // console.log(feature.geometry.coordinates); // → TypeError: Cannot read properties of undefined // ✅ Optional chaining — aman, return undefined jika tidak ada console.log(feature.geometry?.coordinates); // undefined (tidak error) console.log(feature.properties?.NAMA); // "Monas" // Bisa berlapis & dengan method/array console.log(data?.features?.[0]?.properties?.NAMA);

B. Nullish Coalescing (??)

Mirip ||, tapi hanya memberi nilai cadangan jika kiri benar-benar null atau undefined (bukan 0 atau "").

// Perbedaan || vs ?? const jumlah = 0; console.log(jumlah || 10); // 10 ← salah! 0 dianggap falsy console.log(jumlah ?? 10); // 0 ← benar! 0 itu nilai valid // ?? hanya ganti jika null / undefined const rating = 0; const tampil = rating ?? "Belum dinilai"; console.log(tampil); // 0 (rating valid, tetap dipakai)
🛡️ Kombinasi pamungkas untuk data API:
const nama = feature?.properties?.NAMA ?? "Tanpa Nama";
Aman dari error dan punya nilai cadangan. Ini pola yang sangat sering dipakai di WebGIS.

8. Array Methods Modern ES6+

Inti dari mengolah data di JavaScript modern. Semua menerima callback (biasanya arrow function) dan tidak mengubah array asli (kecuali sort).

MethodFungsiMengembalikan
map()Ubah setiap itemArray baru (ukuran sama)
filter()Saring item sesuai kondisiArray baru (lebih kecil)
find()Cari 1 item pertama yang cocok1 item (atau undefined)
reduce()Gabungkan jadi 1 nilai1 nilai (angka/object/dll)
some()Cek apakah ADA yang cocoktrue / false
every()Cek apakah SEMUA cocoktrue / false
sort()Urutkan arrayArray terurut (ubah asli!)

A. map() — Transformasi Data

const angka = [1, 2, 3, 4]; const kuadrat = angka.map(n => n * n); console.log(kuadrat); // [1, 4, 9, 16] // Contoh WebGIS: ambil hanya nama dari array lokasi const lokasi = [ { nama: "Monas", kota: "Jakarta" }, { nama: "Dago", kota: "Bandung" } ]; const namaSaja = lokasi.map(loc => loc.nama); console.log(namaSaja); // ["Monas", "Dago"]

B. filter() — Menyaring Data

const angka = [5, 12, 8, 30, 3]; const besar = angka.filter(n => n > 10); console.log(besar); // [12, 30] // Contoh WebGIS: hanya lokasi yang BUKA const bukaSaja = lokasi.filter(loc => loc.status === "BUKA");

C. find() — Mencari Satu Item

const lokasi = [ { id: 1, nama: "Monas" }, { id: 2, nama: "Ancol" } ]; const hasil = lokasi.find(loc => loc.id === 2); console.log(hasil); // { id: 2, nama: "Ancol" }

D. reduce() — Menggabungkan Jadi Satu Nilai

// Struktur: reduce((akumulator, item) => ..., nilaiAwal) const angka = [10, 20, 30]; const total = angka.reduce((acc, n) => acc + n, 0); console.log(total); // 60 // Langkah: acc=0+10 → 10, acc=10+20 → 30, acc=30+30 → 60

E. some() & every()

const nilai = [80, 90, 65]; console.log(nilai.some(n => n < 70)); // true — ada yang di bawah 70 console.log(nilai.every(n => n >= 70)); // false — tidak semua >= 70

F. Method Chaining (Menggabungkan)

// Kekuatan asli array methods: bisa dirangkai! const lokasi = [ { nama: "Monas", kota: "Jakarta", status: "BUKA" }, { nama: "Dago", kota: "Bandung", status: "TUTUP" }, { nama: "Ancol", kota: "Jakarta", status: "BUKA" } ]; const hasil = lokasi .filter(loc => loc.status === "BUKA") // hanya yang BUKA .map(loc => loc.nama); // ambil namanya saja console.log(hasil); // ["Monas", "Ancol"]
✅ Inti pengolahan data WebGIS: Ambil GeoJSON → filter() lokasi sesuai kriteria → map() jadi format yang dibutuhkan → tampilkan ke DOM. Method chaining membuat kode sangat ringkas.

9. Object Methods & Shorthand ES6+

A. Property Shorthand

const nama = "Monas"; const kota = "Jakarta"; // Cara lama const lokasi = { nama: nama, kota: kota }; // Cara modern — jika nama key = nama variabel, cukup tulis sekali const lokasi = { nama, kota }; console.log(lokasi); // { nama: "Monas", kota: "Jakarta" }

B. Object.keys / values / entries

const lokasi = { nama: "Monas", kota: "Jakarta", status: "BUKA" }; console.log(Object.keys(lokasi)); // ["nama", "kota", "status"] console.log(Object.values(lokasi)); // ["Monas", "Jakarta", "BUKA"] console.log(Object.entries(lokasi)); // [["nama","Monas"], ["kota","Jakarta"], ...] // Loop semua property object Object.entries(lokasi).forEach(([key, value]) => { console.log(`${key}: ${value}`); }); // nama: Monas // kota: Jakarta // status: BUKA
💡 Object.entries() + destructuring [key, value] sangat berguna untuk menampilkan semua properti sebuah feature GeoJSON secara otomatis (misal di popup peta).

10. Modules (import / export) ES6

Module memungkinkan kita memecah kode menjadi file-file terpisah yang saling terhubung. Membuat kode lebih rapi dan mudah dikelola.

A. Named Export

// === file: utils.js === export const PI = 3.14; export function hitungJarak(a, b) { return Math.abs(a - b); } // === file: main.js === import { PI, hitungJarak } from "./utils.js"; console.log(PI); // 3.14 console.log(hitungJarak(10, 3)); // 7

B. Default Export

// === file: peta.js === export default function buatPeta() { return "Peta dibuat!"; } // === file: main.js === // default export — nama saat import bebas, tanpa { } import buatPeta from "./peta.js"; console.log(buatPeta()); // "Peta dibuat!"
// Di HTML, script module ditandai type="module" <script type="module" src="main.js"></script>
📦 Library modern seperti Leaflet, Firebase, dan framework seperti React semuanya memakai sistem module ini. Kamu sudah melihatnya di config Firebase: import { initializeApp } from "...".

11. Promise & async/await (Lanjutan) ES6+

Di Sesi 8 kita sudah pakai async/await untuk fetch. Berikut pemahaman lebih dalam tentang Promise yang ada di baliknya.

A. Apa itu Promise?

Promise adalah "janji" hasil dari operasi asynchronous. Punya 3 status: pending (menunggu), fulfilled (berhasil), rejected (gagal).

// fetch() mengembalikan sebuah Promise const promise = fetch(url); console.log(promise); // Promise { <pending> } // .then() = saat berhasil, .catch() = saat gagal fetch(url) .then(response => response.json()) .then(data => console.log(data)) .catch(error => console.error(error));

B. async/await (Cara Modern)

async function ambilLokasi() { try { const response = await fetch(url); const data = await response.json(); // destructuring + optional chaining + array methods (gabungan ES6+!) const nama = data?.features ?.filter(f => f.properties?.NAMA) .map(f => f.properties.NAMA); console.log(nama); } catch (error) { console.error("Gagal:", error); } }

C. Promise.all — Beberapa Request Sekaligus

// Jalankan beberapa fetch BERSAMAAN (lebih cepat!) async function ambilSemua() { const [lokasi, cuaca] = await Promise.all([ fetch(urlLokasi).then(r => r.json()), fetch(urlCuaca).then(r => r.json()) ]); console.log(lokasi, cuaca); // kedua data sudah siap } // Tanpa Promise.all: harus menunggu satu per satu (lebih lambat)
✅ Rangkuman: Modern JavaScript membuat kode WebGIS lebih ringkas dan aman. Gabungan async/await + destructuring + optional chaining + array methods adalah pola standar untuk mengambil dan mengolah data peta dari API.
