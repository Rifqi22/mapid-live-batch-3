1. Pengetahuan Dasar Website

Sebelum belajar HTML, penting untuk memahami konsep-konsep dasar ekosistem web berikut ini:

Istilah Penjelasan
Internet Komunikasi antara dua komputer atau lebih menggunakan jaringan kabel/wifi untuk bertukar data.
Web Kumpulan informasi di internet berupa teks, gambar, audio, video, dll.
Web Server Komputer/aplikasi yang menyimpan dan mengirimkan isi website ke pengguna ketika diminta.
Web Browser Aplikasi untuk mengakses dan menampilkan website (Chrome, Firefox, Edge, Safari).
IP Address Alamat unik setiap perangkat di internet. Contoh: 8.8.8.8
Port Jalur komunikasi pada IP Address. Contoh: HTTP = port 80, HTTPS = port 443.
Domain Nama mudah diingat pengganti IP Address. Contoh: 8.8.8.8 → google.com
Repository / Repo Tempat penyimpanan project berisi kode, aset, dan riwayat perubahan (Git, GitHub).
GIT Version control system untuk mengelola perubahan kode dari waktu ke waktu.
Deploy Proses mengirimkan aplikasi dari lokal ke server agar bisa diakses pengguna via internet.
Code / Text Editor Aplikasi untuk menulis dan mengedit kode (VS Code, Sublime Text, Notepad++).
Localhost Alamat untuk menjalankan aplikasi di komputer sendiri sebelum dipublikasikan (127.0.0.1).
Live Server Fitur VS Code untuk menjalankan website lokal yang otomatis refresh saat kode diubah.
Terminal Jendela perintah teks untuk menjalankan perintah sistem atau tools developer.
Komentar Teks penanda pada kode yang tidak dieksekusi browser. Di HTML: <!-- komentar -->
Module File/unit kode yang bisa diimpor dan digunakan di tempat lain.
Library Kumpulan kode/fungsi siap pakai untuk mempermudah pengembangan (misal: Leaflet.js untuk peta).
Framework Kerangka kerja lengkap yang mengatur cara kode ditulis dan dijalankan (misal: Bootstrap, React).

Tiga Bahasa Utama Pembuatan Website

  • HTML — Struktur/kerangka halaman (skeleton)
  • CSS — Tampilan/desain halaman (skin)
  • JavaScript — Interaksi & logika (brain)
💡 Analogi: HTML adalah tulang (skeleton), CSS adalah kulit & pakaian (skin), dan JavaScript adalah otak (brain) dari sebuah website.

2. Pengenalan HTML

HTML (HyperText Markup Language) adalah bahasa standar untuk membuat dan menyusun halaman website. HTML bukan bahasa pemrograman, melainkan bahasa markup yang memberi tanda (tag) pada konten agar browser tahu cara menampilkannya.

Fungsi HTML dalam Website

  1. Membuat struktur halaman (header, body, footer)
  2. Menampilkan konten (teks, gambar, video, audio)
  3. Menyediakan navigasi melalui link (anchor)
  4. Menjadi fondasi utama yang dikombinasikan dengan CSS dan JavaScript

Pengetahuan Dasar HTML: Element, Tag, Attribute

KonsepPengertianContoh
Element Keseluruhan bagian HTML: tag pembuka + isi + tag penutup <p>Teks</p>
Tag Penanda awal (<tag>) dan akhir (</tag>) elemen <h1> ... </h1>
Attribute Informasi tambahan di dalam tag pembuka (nama="nilai") <a href="url">

Aturan Penulisan HTML

  • Tag tidak case-sensitive, tapi sangat disarankan huruf kecil semua (<div> bukan <DIV>).
  • Tag harus ditulis benar: ada opening tag dan closing tag.
  • Nilai atribut harus diapit tanda petik ("" atau '').
  • Elemen harus tersusun rapi (Nesting) — tidak boleh saling tumpang tindih. Benar: <i><b>teks</b></i> | Salah: <i><b>teks</i></b>
  • Gunakan elemen sesuai fungsinya (Semantic HTML).

3. Struktur HTML

Setiap file HTML yang valid harus memiliki struktur berikut:

<!-- Deklarasi DOCTYPE: memberitahu browser ini adalah dokumen HTML5 --> <!DOCTYPE html> <html lang="id"> <!-- Elemen root pembungkus semua HTML --> <head> <!-- Metadata, tidak tampil ke pengguna --> <meta charset="UTF-8"> <meta name="viewport" content="width=device-width, initial-scale=1.0"> <title>Judul Halaman</title> </head> <body> <!-- Konten utama yang tampil ke pengguna --> <!-- Semua konten ditulis di sini --> </body> </html>
TagFungsi
<!DOCTYPE html>Deklarasi HTML5 — harus selalu ada di baris pertama
<html>Elemen utama pembungkus seluruh isi dokumen
<head>Metadata: judul, encoding, viewport, link CSS, script
<title>Judul halaman di tab browser & hasil pencarian
<body>Konten yang ditampilkan ke pengguna

4. Tag HTML Text

Heading — <h1> sampai <h6>

<h1> adalah judul utama halaman. Penting untuk SEO — mesin pencari membaca <h1> untuk memahami topik. <h2>–<h6> digunakan untuk subjudul dan sub-subjudul.

h1 — Judul Utama (1x per halaman)

h2 — Judul Bagian Besar

h3 — Sub Bagian

h4 — Sub-sub Bagian

h5 — Level 5
h6 — Level 6 (paling kecil)

Paragraf — <p>

Tag <p> digunakan untuk menulis paragraf teks. Browser otomatis menambahkan jarak (margin) di atas dan bawah paragraf.

Ini adalah paragraf kedua. Perhatikan ada jarak dengan paragraf di atasnya.

Line Break — <br>

Tag <br> memaksa baris baru tanpa membuat paragraf baru.

Horizontal Rule — <hr>

5. Tag HTML Table

Tabel digunakan untuk menampilkan data dalam bentuk baris dan kolom.

Anatomi Tabel HTML

Tag Fungsi
<table>Wadah/pembungkus seluruh tabel
<thead>Bagian header tabel (membungkus baris judul kolom)
<tbody>Bagian isi/data utama tabel
<tfoot>Bagian footer tabel (total, catatan, dll.)
<tr>Table Row — satu baris dalam tabel
<th>Table Header — sel judul kolom (tebal & centered)
<td>Table Data — sel isi data biasa

Contoh Tabel: Data Transportasi DKI Jakarta

No Kode Nama Transportasi Jumlah Rute
1 TJ Transjakarta 50
2 MRT Mass Rapid Transit 2

4 KRL Kereta Rel Listrik 5

💡 Atribut Khusus Tabel:
• colspan="n" — sel menyatu secara horizontal (n kolom)
• rowspan="n" — sel menyatu secara vertikal (n baris)

8. Tag HTML Kelompok (Container) & Semantik HTML

Tag kelompok mengelompokkan konten agar struktur lebih jelas, rapi, mudah dibaca, dan ramah SEO serta aksesibilitas.

TagFungsi UtamaSemantik
<header>Bagian atas / judul halaman atau seksi✅ Ya
<nav>Menu navigasi✅ Ya
<main>Konten utama halaman (1x per halaman)✅ Ya
<section>Bagian konten berdasarkan topik/tema✅ Ya
<article>Konten mandiri (blog post, berita, komentar)✅ Ya
<aside>Konten sampingan / sidebar✅ Ya
<figure>Gambar / ilustrasi dengan keterangan✅ Ya
<figcaption>Keterangan/caption dari <figure>✅ Ya
<footer>Bagian bawah/penutup halaman atau seksi✅ Ya
<div>Wadah umum untuk layout (tidak semantik)❌ Tidak
<span>Wadah inline, untuk styling teks per bagian❌ Tidak

Atribut HTML Penting

AtributDigunakan di TagFungsi

✅ Mengapa Semantik HTML penting?
1. SEO: Mesin pencari lebih mudah memahami struktur konten.
2. Aksesibilitas: Screen reader membantu pengguna dengan disabilitas navigasi.
3. Keterbacaan Kode: Lebih mudah dipahami oleh developer lain.

9. Materi Tambahan Penting Bonus

Materi-materi berikut tidak tercakup di slide utama, namun sangat penting untuk dipelajari dalam pengembangan WebGIS.

9.1 Tag Link — <a> (Anchor) Bonus

Tag <a> digunakan untuk membuat hyperlink — tautan ke halaman lain, bagian halaman yang sama, file, email, atau telepon.

<!-- Link ke halaman lain (buka di tab baru) --> <a href="https://mapid.co.id" target="_blank">Kunjungi MAPID</a> <!-- Link ke bagian di halaman yang sama (anchor in-page) --> <a href="#bagian-peta">Lihat Peta</a> <!-- Link email (membuka aplikasi email) --> <a href="mailto:info@mapid.co.id">Kirim Email</a> <!-- Link telepon (berguna di mobile) --> <a href="tel:+6281234567890">Hubungi Kami</a> <!-- Link download file --> <a href="peta.pdf" download>Download Peta PDF</a>

Contoh link aktif: 🌐 Website MAPID  |  ↑ Kembali ke Atas

9.2 Tag Gambar — <img> Bonus

Tag <img> digunakan untuk menampilkan gambar. Ini adalah void element — tidak perlu tag penutup.

<!-- Gambar dari internet (URL) --> <img src="https://via.placeholder.com/400x200" alt="Deskripsi gambar untuk aksesibilitas & SEO" width="400" height="200" title="Tooltip saat di-hover" > <!-- Gambar lokal --> <img src="assets/peta-jakarta.png" alt="Peta Jakarta"> <!-- figure + figcaption: gambar dengan keterangan (semantik) --> <figure> <img src="peta.jpg" alt="Peta Transportasi DKI"> <figcaption>Gambar 1. Peta Transportasi DKI Jakarta 2026</figcaption> </figure>

9.3 Tag Embed / Iframe — <iframe> Bonus — Sangat Penting untuk WebGIS!

Tag <iframe> menyisipkan konten dari halaman lain (peta, video YouTube, Google Maps, dll.) langsung ke dalam halaman kita.

<!-- Menyisipkan peta dari MAPID (contoh umum) --> <iframe src="https://mapid.co.id/map/embed/ID_PETA" width="100%" height="400" style="border:0;" allowfullscreen loading="lazy" title="Peta WebGIS" ></iframe> <!-- Menyisipkan Google Maps --> <iframe src="https://www.google.com/maps/embed?pb=..." width="600" height="450" style="border:0;" allowfullscreen loading="lazy" ></iframe>

Contoh Iframe Peta OpenStreetMap — Jakarta:

↑ Peta interaktif OpenStreetMap area DKI Jakarta (embedded via <iframe>)

9.4 Tag Video & Audio Bonus

HTML5 memiliki tag native untuk menampilkan video dan audio tanpa plugin tambahan.

<!-- Tag Video --> <video width="640" height="360" controls poster="thumbnail.jpg"> <!-- <source> untuk berbagai format agar kompatibel di semua browser --> <source src="video.mp4" type="video/mp4"> <source src="video.webm" type="video/webm"> Browser Anda tidak mendukung tag video. </video> <!-- Tag Audio --> <audio controls> <source src="audio.mp3" type="audio/mpeg"> Browser Anda tidak mendukung tag audio. </audio>
📌 Atribut video/audio:
• controls : tampilkan tombol play/pause/volume
• autoplay : putar otomatis (perlu muted di banyak browser)
• loop : putar berulang
• muted : mulai dalam keadaan bisu
• poster : gambar thumbnail sebelum video diputar (khusus video)

9.5 Meta Tags Penting di <head> Bonus — SEO & Responsif

Meta tag memberikan informasi tentang halaman ke browser, mesin pencari, dan media sosial.

<head> <!-- Encoding karakter (wajib ada) --> <meta charset="UTF-8"> <!-- Responsif di mobile (wajib ada) --> <meta name="viewport" content="width=device-width, initial-scale=1.0"> <!-- Deskripsi untuk Google (150-160 karakter) --> <meta name="description" content="WebGIS peta transportasi DKI Jakarta"> <!-- Open Graph: tampilan saat link dibagikan ke media sosial --> <meta property="og:title" content="WebGIS Transportasi Jakarta"> <meta property="og:description" content="Peta interaktif transportasi DKI"> <meta property="og:image" content="thumbnail.jpg"> <!-- Menghubungkan file CSS eksternal --> <link rel="stylesheet" href="style.css"> <!-- Favicon (ikon kecil di tab browser) --> <link rel="icon" type="image/png" href="favicon.png"> </head>

9.6 Description List — <dl> Bonus

Jenis list ketiga yang sering terlupakan. Cocok untuk pasangan istilah-definisi, seperti glossary.

<dl> <dt>WebGIS</dt> <!-- dt = Description Term (istilah) --> <dd>Sistem Informasi Geografis berbasis web</dd> <!-- dd = Description Data (definisi) --> <dt>GeoJSON</dt> <dd>Format data geografis berbasis JSON untuk menyimpan fitur spasial</dd> </dl>
WebGIS
Sistem Informasi Geografis berbasis web yang memungkinkan visualisasi dan analisis data spasial melalui browser.
GeoJSON
Format data geografis berbasis JSON untuk menyimpan fitur spasial seperti titik, garis, dan poligon.

9.7 Menyisipkan JavaScript — <script> Bonus

Tag <script> menyisipkan kode JavaScript ke halaman HTML.

<!-- JavaScript Internal (langsung di HTML) --> <script> console.log("Halo dari JavaScript!"); alert("WebGIS siap dibangun!"); </script> <!-- JavaScript Eksternal (file .js terpisah) --> <script src="script.js"></script> <!-- Menyisipkan library Leaflet.js (CDN) untuk peta --> <script src="https://unpkg.com/leaflet@1.9.4/dist/leaflet.js"></script>
⚠️ Best Practice: Tempatkan tag <script> di bagian bawah <body> (sebelum </body>) agar halaman tampil lebih cepat dan tidak memblokir rendering HTML. Atau gunakan atribut defer pada tag script di <head>.
