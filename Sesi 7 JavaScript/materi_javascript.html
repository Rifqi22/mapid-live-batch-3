1. Pengenalan JavaScript Sesi 7

JavaScript adalah bahasa pemrograman tingkat tinggi yang digunakan untuk membuat website menjadi interaktif dan dinamis. JavaScript berjalan di browser menggunakan JavaScript Engine (seperti V8 di Chrome).

Analogi: HTML, CSS, JavaScript

TeknologiPeranAnalogi Rumah
HTMLStruktur / kontenKerangka & dinding rumah
CSSTampilan / gayaCat, dekorasi, tata letak
JavaScriptInteraksi / perilakuListrik, saklar lampu, pintu otomatis

Fungsi Utama JavaScript

  • Mengontrol interaksi (klik tombol, form, animasi)
  • Memanipulasi HTML & CSS (ubah tampilan tanpa reload halaman)
  • Mengolah data (validasi form, kalkulasi, filter)
  • Mengambil data dari server/API (menampilkan data peta, cuaca, dll)

2 Cara Penulisan JavaScript

MetodeCaraKapan Digunakan
Internal JS Tag <script> di dalam file HTML Belajar, file kecil
External JS File .js terpisah via <script src="..."> ✅ Proyek nyata, best practice
// ========== 1. Internal JavaScript ========== // Tulis langsung di dalam tag <script> pada file HTML <script> console.log("Hello World"); alert("Selamat datang!"); </script> // ========== 2. External JavaScript ========== // Buat file terpisah: script.js // Lalu hubungkan di HTML: <script src="script.js"></script>
⚠️ Letakkan <script> sebelum </body> (di bagian bawah HTML). Ini memastikan semua elemen HTML sudah dimuat sebelum JavaScript dijalankan.

Template Literal (Backtick)

Cara modern menyisipkan variabel ke dalam string — menggunakan backtick ` dan ${}:

const nama = "Rifqi"; const umur = 21; // Cara lama: concatenation (+) console.log("Nama saya " + nama + ", umur " + umur + " tahun."); // Cara modern: template literal (lebih rapi!) console.log(`Nama saya ${nama}, umur ${umur} tahun.`);
✅ Tips: Selalu gunakan template literal (backtick) untuk menggabungkan string dengan variabel. Lebih mudah dibaca dan tidak rawan error tanda kutip.

3. Variable Sesi 7

Variable adalah tempat menyimpan data. Seperti "kotak berlabel" yang berisi nilai tertentu.

JenisBisa Diubah?ScopeRekomendasi
var✅ YaFunction scope❌ Jangan dipakai (cara lama)
let✅ YaBlock scope {}✅ Untuk nilai yang berubah
const❌ TidakBlock scope {}✅ Default, untuk nilai tetap
// Anatomi: (jenis variable) + (nama) + (=) + (nilai) var namaDepan = "Rifqi"; // ❌ hindari var let namaBelakang = "Naufal"; // ✅ bisa diubah nanti const namaLengkap = "Rifqi Naufal"; // ✅ tidak bisa diubah // let bisa diubah nilainya: namaBelakang = "Alfayyadh"; // ✅ OK // const TIDAK bisa diubah: // namaLengkap = "Budi"; ← ERROR! Assignment to constant variable // Tapi const ARRAY & OBJECT isinya bisa diubah: const rute = ["TJ", "MRT"]; rute.push("LRT"); // ✅ OK — menambah isi array // rute = []; ← ERROR! — tidak bisa ganti variabel-nya

Aturan Penamaan Variable

  • Harus dimulai dengan huruf, _, atau $ (tidak boleh angka)
  • Tidak boleh pakai spasi — gunakan camelCase: namaDepan, jumlahRute
  • Tidak boleh pakai reserved word: let, const, function, return, dll
  • Case-sensitive: nama ≠ Nama ≠ NAMA
💡 Best practice: Default pakai const. Hanya pakai let kalau nilainya memang perlu diubah. Jangan pernah pakai var.

5. Operator Sesi 7

A. Operator Aritmatika

const a = 10; const b = 3; console.log(a + b); // 13 — penjumlahan console.log(a - b); // 7 — pengurangan console.log(a * b); // 30 — perkalian console.log(a / b); // 3.33 — pembagian console.log(a % b); // 1 — sisa bagi (modulus) console.log(a ** b); // 1000 — pangkat (10³) // Increment & Decrement let count = 5; count++; // 6 — tambah 1 count--; // 5 — kurang 1

B. Operator Perbandingan

console.log(10 > 5); // true — lebih besar console.log(10 < 5); // false — lebih kecil console.log(10 >= 10); // true — lebih besar atau sama console.log(10 <= 5); // false — lebih kecil atau sama // ⚠️ Perbedaan penting: == vs === console.log(5 == "5"); // true — hanya bandingkan NILAI (type coercion) console.log(5 === "5"); // false — bandingkan NILAI + TIPE DATA console.log(5 === 5); // true — nilai sama, tipe sama console.log(5 != "5"); // false — nilai sama console.log(5 !== "5"); // true — tipe berbeda
⚠️ Selalu pakai === (strict equality), bukan ==. Ini mencegah bug tersembunyi dari type coercion. Contoh: 0 == "" hasilnya true (padahal seharusnya false).

C. Operator Logika

const hujan = true; const dingin = false; // && (AND) — true jika SEMUA kondisi true console.log(hujan && dingin); // false (satu false = false) // || (OR) — true jika SALAH SATU kondisi true console.log(hujan || dingin); // true // ! (NOT) — membalik nilai console.log(!hujan); // false (kebalikan dari true) // Contoh kombinasi: const nilai = 85; const hadir = true; if (nilai >= 70 && hadir) { console.log("Lulus!"); // ← ini yang jalan }

7. Percabangan Sesi 7

Percabangan digunakan untuk membuat program mengambil keputusan berdasarkan kondisi tertentu.

A. if / else if / else

const nilai = 75; if (nilai >= 80) { console.log("Nilai A — Sangat Baik"); } else if (nilai >= 70) { console.log("Nilai B — Baik"); // ← ini yang jalan } else if (nilai >= 60) { console.log("Nilai C — Cukup"); } else { console.log("Tidak Lulus"); } // Contoh WebGIS: cek jarak const jarakKm = 2.5; if (jarakKm < 1) { console.log("Lokasi sangat dekat — jalan kaki saja"); } else if (jarakKm < 5) { console.log("Lokasi cukup dekat — naik Transjakarta"); // ← ini } else { console.log("Lokasi jauh — naik KRL atau MRT"); }

B. switch / case

const kode = "MRT"; switch (kode) { case "TJ": console.log("Transjakarta — 50 rute"); break; case "MRT": console.log("MRT Jakarta — 2 rute"); // ← ini yang jalan break; case "LRT": console.log("LRT Jakarta — 1 rute"); break; case "KRL": console.log("KRL Commuter — 5 rute"); break; default: console.log("Kode tidak dikenal"); }
💡 Kapan pakai apa? if/else untuk kondisi rentang nilai (>, <, >=). switch/case untuk mencocokkan nilai tetap (===).

8. Perulangan (Looping) Sesi 7

Perulangan mengulang perintah berkali-kali secara otomatis tanpa harus menulis kode berulang.

A. for loop

Digunakan saat kita tahu jumlah perulangannya.

// Struktur: for (mulai; kondisi; increment) for (let i = 0; i < 5; i++) { console.log(`Nilai ke-${i}`); } // Output: Nilai ke-0, Nilai ke-1, ... Nilai ke-4 // Looping array dengan for: const moda = ["TJ", "MRT", "LRT", "KRL"]; for (let i = 0; i < moda.length; i++) { console.log(`${i + 1}. ${moda[i]}`); }

B. for...of

Cara lebih simpel untuk membaca isi array tanpa index.

const lokasi = ["Monas", "Ancol", "Bandung"]; for (let item of lokasi) { console.log(`Lokasi: ${item}`); } // Lokasi: Monas // Lokasi: Ancol // Lokasi: Bandung

C. while loop

Mengulang selama kondisi masih true. Hati-hati infinite loop!

let i = 0; while (i < 3) { console.log(`Marker ke-${i}`); i++; // WAJIB! tanpa ini = infinite loop }

D. do...while

Mirip while, tapi minimal dijalankan 1 kali sebelum cek kondisi.

let j = 0; do { console.log(`Marker ke-${j}`); j++; } while (j < 3);

break & continue

// break — hentikan loop sepenuhnya for (let i = 0; i < 10; i++) { if (i === 5) break; // stop di angka 5 console.log(i); // Output: 0, 1, 2, 3, 4 } // continue — lewati 1 putaran, lanjut ke berikutnya for (let i = 0; i < 5; i++) { if (i === 2) continue; // lewati angka 2 console.log(i); // Output: 0, 1, 3, 4 }

9. Function Sesi 7

Function adalah sekumpulan kode yang dibungkus dalam satu nama dan bisa dipanggil berkali-kali. Seperti "resep masakan" — ditulis sekali, bisa dipakai kapan saja.

Struktur Function

BagianPenjelasan
Nama functionIdentitas untuk memanggil function
Parameter (input)Data yang diterima function untuk diproses
Proses / isiKode yang dijalankan di dalam function
return (output)Nilai yang dikembalikan function

A. Function Declaration

// Membuat function function sapa(nama) { return `Halo, ${nama}! Selamat belajar WebGIS.`; } // Memanggil function const pesan = sapa("Rifqi"); console.log(pesan); // "Halo, Rifqi! Selamat belajar WebGIS." // Function dengan parameter & proses function hitungLuas(panjang, lebar) { const luas = panjang * lebar; return luas; } console.log(hitungLuas(10, 5)); // 50 console.log(hitungLuas(7, 3)); // 21

B. Function Expression (Arrow Function)

// Function disimpan dalam variabel const tambah = function(a, b) { return a + b; }; // Arrow function — cara singkat menulis function const kali = (a, b) => { return a * b; }; // Arrow function super singkat (1 baris langsung return) const kuadrat = (n) => n * n; console.log(tambah(3, 4)); // 7 console.log(kali(5, 6)); // 30 console.log(kuadrat(8)); // 64
💡 Function Declaration bisa dipanggil sebelum ditulis (hoisting). Arrow Function tidak bisa — harus ditulis dulu baru dipanggil.

10. DOM (Document Object Model) Sesi 8

DOM adalah representasi struktur HTML dalam bentuk object/pohon (tree) yang bisa diakses dan dimanipulasi oleh JavaScript. Dengan DOM, kamu bisa mengubah tampilan halaman tanpa reload.

Bagaimana DOM Bekerja?

  1. Browser membaca file HTML
  2. Browser membangun DOM Tree (pohon elemen)
  3. JavaScript mengakses DOM Tree dan memanipulasi elemen-elemen di dalamnya

Visualisasi DOM Tree

Setiap tag HTML menjadi "node" (simpul) di pohon DOM:

Event Handling (addEventListener)

Event adalah aksi yang terjadi di halaman (klik, ketik, hover, submit, dll). Kita bisa merespon event menggunakan addEventListener.

EventKapan TerjadiContoh
"click"Saat elemen diklikTombol, link
"submit"Saat form disubmitForm pendaftaran
"input"Saat user mengetik di inputSearch box
"change"Saat nilai input berubahDropdown, checkbox
"mouseover"Saat mouse hover di elemenTooltip, highlight
"keydown"Saat tombol keyboard ditekanShortcut
// === KLIK TOMBOL === // HTML: <button id="btn">Klik Saya</button> const tombol = document.getElementById("btn"); tombol.addEventListener("click", function() { alert("Tombol diklik!"); }); // === FORM SUBMIT === // HTML: <form class="register">...</form> const form = document.querySelector(".register"); form.addEventListener("submit", function(e) { e.preventDefault(); // ← PENTING! mencegah halaman refresh const nama = document.getElementById("namaDepan").value; const email = document.getElementById("email").value; console.log(`Nama: ${nama}, Email: ${email}`); alert(`Data berhasil dikirim!\nNama: ${nama}`); }); // === INPUT REAL-TIME === // HTML: <input id="search" type="text"> const search = document.getElementById("search"); search.addEventListener("input", function(e) { console.log("User mengetik:", e.target.value); });
⚠️ e.preventDefault() wajib dipakai pada event "submit" agar halaman tidak refresh. Tanpa ini, data yang sudah diproses akan hilang karena halaman reload.

Event Object (e) — Informasi Tentang Event

Apa itu API?

API (Application Programming Interface) adalah mekanisme yang memungkinkan dua sistem berkomunikasi. REST API menggunakan HTTP untuk bertukar data — biasanya format JSON.

Alur Kerja

JavaScript (Frontend) → Kirim Request ke API → Server memproses → Server kirim Response (JSON) → JavaScript tampilkan ke DOM

Struktur REST API

KomponenPenjelasanContoh
Endpoint (URL)Alamat API di serverhttps://geoserver.mapid.io/layers_new/get_layer
HTTP MethodAksi yang dilakukanGET, POST, PUT, DELETE
Query ParameterInfo tambahan di URL (setelah ?)?api_key=...&layer_id=...
HeadersInfo tambahan (format data, auth)Content-Type: application/json
BodyData yang dikirim (POST/PUT){ "nama": "Rifqi" }
ResponseBalasan dari serverStatus code + data JSON

Contoh API Nyata: MAPID GeoServer

Sepanjang materi ini kita akan memakai API MAPID GeoServer yang menyediakan data spasial (lokasi) dalam format GeoJSON:

// Endpoint MAPID GeoServer — mengambil 1 layer data lokasi https://geoserver.mapid.io/layers_new/get_layer ?api_key=7b8019aa264248e89e3fd5b27253132f // kunci akses API &layer_id=69ace1b8643f7636a769ce7c // ID layer yang diambil &project_id=69ab9fb96c69e6252868efaf // ID project // Bagian setelah tanda "?" disebut QUERY PARAMETER // Dipisah dengan "&", formatnya: nama=nilai
🗺️ Apa itu GeoJSON? Format JSON khusus untuk data geografis. Berisi kumpulan features (titik/garis/area), masing-masing punya geometry (koordinat) dan properties (info seperti nama, alamat). Ini format standar di dunia WebGIS.

NASUTION NO.117...", "KABKOT": "KOTA BANDUNG", "KECAMATAN": "UJUNGBERUNG", "STATUS": "BUKA" } } // ... dan seterusnya (total 67 lokasi) ] }

Synchronous vs Asynchronous

Sebelum belajar fetch, kamu perlu paham perbedaan dua cara JavaScript menjalankan kode:

AspekSynchronous (Sync)Asynchronous (Async)
Cara kerja Dijalankan berurutan, satu per satu Bisa berjalan bersamaan, tidak perlu menunggu
Jika ada proses lama Kode berikutnya harus menunggu (blocking) Kode berikutnya tetap jalan (non-blocking)
Analogi Antri di kasir — harus selesai satu baru lanjut Pesan makanan — sambil menunggu, bisa ngobrol
Contoh console.log(), kalkulasi, loop fetch(), setTimeout(), baca file

Synchronous — Berurutan

// Synchronous: kode dijalankan SATU PER SATU dari atas ke bawah console.log("1. Mulai"); console.log("2. Proses data"); console.log("3. Selesai"); // Output (berurutan): // 1. Mulai // 2. Proses data // 3. Selesai

Asynchronous — Tidak Menunggu

// Asynchronous: proses yang butuh waktu TIDAK menghalangi kode lain console.log("1. Mulai"); setTimeout(function() { console.log("2. Data selesai dimuat (setelah 2 detik)"); }, 2000); // tunggu 2000ms = 2 detik console.log("3. Kode ini jalan duluan!"); // Output (TIDAK berurutan!): // 1. Mulai // 3. Kode ini jalan duluan! ← tidak menunggu setTimeout // 2. Data selesai dimuat (setelah 2 detik) ← muncul belakangan
⚠️ Kenapa ini penting? fetch() adalah operasi asynchronous — mengambil data dari server butuh waktu (bisa 1 detik, bisa 10 detik). Tanpa async, seluruh halaman akan freeze/macet selama menunggu data. Dengan async, halaman tetap bisa digunakan sambil menunggu.

Kenapa fetch() Butuh .then() atau await?

// ❌ SALAH — fetch() belum selesai, data belum ada! const response = fetch("https://api.example.com/data"); console.log(response); // Promise { <pending> } ← bukan data! // ✅ BENAR — tunggu fetch selesai dengan .then() fetch("https://api.example.com/data") .then(function(response) { return response.json(); }) .then(function(data) { console.log(data); }); // ← data sudah ada // ✅ BENAR — atau pakai async/await (lebih rapi) async function ambilData() { const response = await fetch("https://api.example.com/data"); // tunggu selesai const data = await response.json(); // tunggu konversi JSON console.log(data); // ← data sudah ada }
✅ Ringkasan: Gunakan .then() atau async/await untuk "menunggu" proses async selesai. await artinya "tunggu sampai selesai baru lanjut ke baris berikutnya" — hanya bisa dipakai di dalam function yang ditandai async.

Fetch API — Cara Mengambil Data

Cara 1: .then() / .catch() (Promise)

// GET — Mengambil data lokasi dari MAPID GeoServer const url = "https://geoserver.mapid.io/layers_new/get_layer?api_key=7b8019aa264248e89e3fd5b27253132f&layer_id=69ace1b8643f7636a769ce7c&project_id=69ab9fb96c69e6252868efaf"; fetch(url) .then(function(response) { console.log("Status:", response.status); // 200 return response.json(); // konversi response ke JSON }) .then(function(data) { console.log(data); // object GeoJSON FeatureCollection console.log(data.features); // array berisi 67 lokasi console.log(data.features.length); // 67 }) .catch(function(error) { console.error("Terjadi kesalahan:", error); });

Cara 2: async / await (Lebih Mudah Dibaca)

const url = "https://geoserver.mapid.io/layers_new/get_layer?api_key=7b8019aa264248e89e3fd5b27253132f&layer_id=69ace1b8643f7636a769ce7c&project_id=69ab9fb96c69e6252868efaf"; async function getLokasi() { try { const response = await fetch(url); // Cek apakah response OK (status 200) if (!response.ok) { throw new Error(`HTTP Error: ${response.status}`); } const data = await response.json(); console.log(`Berhasil memuat ${data.features.length} lokasi`); // Ambil lokasi pertama sebagai contoh const pertama = data.features[0]; console.log(pertama.properties.NAMA); // nama lokasi console.log(pertama.geometry.coordinates); // [lng, lat] } catch (error) { console.error("Error:", error); } } getLokasi(); // panggil function
✅ async/await adalah cara modern yang lebih mudah dibaca dibanding .then(). Gunakan try/catch untuk menangani error. Keduanya menghasilkan hal yang sama.

Contoh: GET — Menampilkan Data Lokasi ke Halaman

// HTML: <ul id="daftarLokasi"></ul> const url = "https://geoserver.mapid.io/layers_new/get_layer?api_key=7b8019aa264248e89e3fd5b27253132f&layer_id=69ace1b8643f7636a769ce7c&project_id=69ab9fb96c69e6252868efaf"; async function tampilkanLokasi() { const list = document.getElementById("daftarLokasi"); list.innerHTML = "<li>Memuat data...</li>"; // loading state try { const response = await fetch(url); const data = await response.json(); // Data lokasi ada di dalam data.features (array GeoJSON) const lokasi = data.features; // Kosongkan loading list.innerHTML = ""; // Loop setiap lokasi dan buat elemen <li> lokasi.forEach(function(item) { const nama = item.properties.NAMA || "Tanpa Nama"; const alamat = item.properties.KABKOT || "-"; const li = document.createElement("li"); li.innerText = `${nama} — ${alamat}`; list.appendChild(li); }); } catch (error) { list.innerHTML = "<li style='color:red'>Gagal memuat data!</li>"; console.error(error); } } tampilkanLokasi();
⚠️ Perhatikan || "Tanpa Nama": Tidak semua lokasi punya properti NAMA (ada yang kosong). Operator || memberi nilai cadangan jika data tidak ada, agar tidak muncul undefined di layar.

Contoh: POST — Mengirim Data ke Server

Untuk membuat data baru, gunakan method: "POST" beserta body. Contoh berikut mengirim data ke API latihan (JSONPlaceholder):

async function kirimData() { try { const response = await fetch("https://jsonplaceholder.typicode.com/posts", { method: "POST", headers: { "Content-Type": "application/json" }, body: JSON.stringify({ nama: "Apartemen Baru", kota: "Bandung", koordinat: [107.61, -6.90] }) }); const data = await response.json(); console.log("Data berhasil dikirim:", data); } catch (error) { console.error("Gagal mengirim data:", error); } } kirimData();

JSON.stringify() & JSON.parse()

// JavaScript Object → JSON String (untuk dikirim ke server) const objek = { nama: "Rifqi", umur: 21 }; const jsonString = JSON.stringify(objek); console.log(jsonString); // '{"nama":"Rifqi","umur":21}' // JSON String → JavaScript Object (data dari server) const text = '{"nama":"Andi","kota":"Jakarta"}'; const obj = JSON.parse(text); console.log(obj.nama); // "Andi"

Contoh Lengkap: Tombol + Fetch GET + Render Cards

Contoh paling sering dipakai di WebGIS: klik tombol → ambil data lokasi → tampilkan sebagai card di halaman.

// HTML: // <button id="btnLoad">Muat Data Lokasi</button> // <div id="status"></div> // <div id="hasil"></div> const url = "https://geoserver.mapid.io/layers_new/get_layer?api_key=7b8019aa264248e89e3fd5b27253132f&layer_id=69ace1b8643f7636a769ce7c&project_id=69ab9fb96c69e6252868efaf"; document.getElementById("btnLoad").addEventListener("click", async function() { const status = document.getElementById("status"); const hasil = document.getElementById("hasil"); status.innerText = "⏳ Memuat data..."; hasil.innerHTML = ""; try { const response = await fetch(url); const data = await response.json(); const lokasi = data.features; status.innerText = `✅ Berhasil memuat ${lokasi.length} lokasi`; // Tampilkan hanya yang punya NAMA, ambil 10 pertama lokasi .filter(function(item) { return item.properties.NAMA; }) .slice(0, 10) .forEach(function(item) { const p = item.properties; const [lng, lat] = item.geometry.coordinates; const card = document.createElement("div"); card.classList.add("card"); card.innerHTML = ` <h4>${p.NAMA}</h4> <p>📍 ${p.KABKOT || "-"}, ${p.KECAMATAN || "-"}</p> <p>🏢 ${p.TIPE_2 || "-"} • Status: ${p.STATUS || "-"}</p> <p>🌐 ${lat.toFixed(5)}, ${lng.toFixed(5)}</p> `; hasil.appendChild(card); }); } catch (error) { status.innerHTML = `❌ Gagal memuat: ${error.message}`; } });
✅ Pola ini menggabungkan semua materi Sesi 8: event handling (click), async/await, fetch GET, loading state, array methods (filter, slice, forEach), destructuring koordinat, dan DOM manipulation (createElement, innerHTML, appendChild). Inilah inti dari membangun WebGIS dengan JavaScript!
